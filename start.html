Manual de integração a API

Última atualização: 16/04/2019

Visão Geral & Objetivo

O objetivo desta documentação é orientar o desenvolvedor sobre como integrar com a API (NOME DA API), descrevendo as funcionalidades, os métodos a serem utilizados, listando informações a serem enviadas e recebidas, e provendo exemplos. O mecanismo de integração com (Nome da API) é simples, de modo que apenas conhecimentos intermediários em linguagem de programação para Web, requisições HTTP/HTTPS e manipulação de arquivos JSON, são necessários para implantar a solução (Nome da api) com sucesso. Nesse manual você encontrará a referência sobre todas as operações disponíveis na API REST da API (Nome da api). Estas operações devem ser executadas utilizando um Token e seu IPadress, nos respectivos endpoints dos ambientes:

Instalação

Por questão de segurança, as requisições a API precisam SEMPRE de um token válido e único em todas as requisições. Para solicitar seu primeiro token, execute a requisição "AUTH".

Exemplos de requisições

Requisição AUTH - Privada

A requisição do tipo POST para o serviço de autenticação (Auth), serve para criar uma autenticação via TOKEN para outras requisições. Após enviar a requisição de autenticação utilizando os parâmetros "AppToken", "appPassword", "ipAddress","login" e "userPassword", outras requisições da aplicação serão permitidas para o TOKEN em questão, que será gerado e retornado após a requisição ser executada.

ATENÇÃO!!! Esse TOKEN será válido por até 15 minutos e somente poderá se utilizado uma ÚNICA vez, pois a cada requisição será gerado um novo TOKEN.

Requisição POST de exemplo
{
	"appToken": "CDD3D0B1C293E96933F2311DF45E52446826C1FF3C7EEBD71076761BA5E9FF25",
	"appPassword": "0253BBFC56DC68DB82AEE56127676B60B3F7F5DE2AF72D0FC974673A16313158",
	"login": "heitorlucasferreira-99@delboux.com.br",
	"userPassword": "123",
	"ipAddress": "localhost"
}
PARÂMETRO TIPO TAMANHO OBRIGATÓRIO DESCRIÇÃO
appToken ALFANUMÉRICO 64 SIM Identificação da aplicação
appPassword ALFANUMÉRICO 64 SIM Senha da aplicação
login ALFANUMÉRICO 30 SIM Nome do Usuário cadastrado na aplicação
TOKEN ALFANUMÉRICO 30 SIM Senha do Usuário cadastrado na aplicação
ipAddress ALFANUMÉRICO/td> 100 SIM IP do user que está acessando a aplicação

O TOKEN gerado será válido por até 15 minutos, ou até a sua utilização. Após utilizá-lo, a API retornará os dados da sua requisição + um novo token para usar na próxima requisição.

Requisição AUTH - Pública

A requisição do tipo POST para o serviço de autenticação Pública(Auth), serve para criar uma autenticação via TOKEN para as requisições do tipo PÚBLICA. Após enviar a requisição de autenticação utilizando os parâmetros "AppToken" e "appPassword", outras requisições do tipo PÚBLICA da aplicação serão permitidas para o TOKEN em questão, que será gerado e retornado após a requisição ser executada.

ATENÇÃO!!! Esse TOKEN será válido por até 15 minutos e somente poderá se utilizado uma ÚNICA vez, pois a cada requisição será gerado um novo TOKEN.

Requisição POST de exemplo
{
"appToken": "CDD3D0B1C293E96933F2311DF45E52446826C1FF3C7EEBD71076761BA5E9FF25",
"appPassword": "0253BBFC56DC68DB82AEE56127676B60B3F7F5DE2AF72D0FC974673A16313158"
}
PARÂMETRO TIPO TAMANHO OBRIGATÓRIO DESCRIÇÃO
appToken ALFANUMÉRICO 64 SIM Identificação da aplicação
appPassword ALFANUMÉRICO 64 SIM Senha da aplicação
