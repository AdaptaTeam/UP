Manual de integração a API

Última atualização: 16/04/2019

Visão Geral & Objetivo

O objetivo desta documentação é orientar o desenvolvedor sobre como integrar com a API (NOME DA API), descrevendo as funcionalidades, os métodos a serem utilizados, listando informações a serem enviadas e recebidas, e provendo exemplos. O mecanismo de integração com (Nome da API) é simples, de modo que apenas conhecimentos intermediários em linguagem de programação para Web, requisições HTTP/HTTPS e manipulação de arquivos JSON, são necessários para implantar a solução (Nome da api) com sucesso. Nesse manual você encontrará a referência sobre todas as operações disponíveis na API REST da API (Nome da API). Estas operações devem ser executadas utilizando um Token e seu IPadress, nos respectivos endpoints dos ambientes:

Instalação

Por questão de segurança, as requisições a API precisam SEMPRE de um token válido e único em todas as requisições. Para solicitar seu primeiro token, execute a requisição "AUTH".

Para executar uma operação, combine a URL base do ambiente com a URL da operação desejada e envie utilizando o verbo HTTP conforme descrito na operação.

SANDBOX PRODUÇÃO
http://adi.adaptaconsultoria.com.br:teste/cashback/api/ http://adi.adaptaconsultoria.com.br:8202/cashback/api/

Exemplos de requisições

POST

/auth/

Requisição AUTH - Privada

A requisição do tipo POST para o serviço de autenticação Privada (Auth) serve para criar uma autenticação via TOKEN para outras requisições do tipo PRIVADA. Após enviar essa requisição utilizando os parâmetros "AppToken", "appPassword", "ipAddress", "login" e "userPassword", outras requisições da aplicação serão permitidas para o TOKEN em questão, que será gerado e retornado após a requisição ser executada.

ATENÇÃO!!! Esse TOKEN será válido por até 15 minutos e somente poderá se utilizado uma ÚNICA vez, pois a cada requisição será gerado um novo TOKEN.

Requisição POST de exemplo
{
	"appToken": "CDD3D0B1C293E96933F2311DF45E52446826C1FF3C7EEBD71076761BA5E9FF25",
	"appPassword": "0253BBFC56DC68DB82AEE56127676B60B3F7F5DE2AF72D0FC974673A16313158",
	"login": "heitorlucasferreira-99@delboux.com.br",
	"userPassword": "123",
	"ipAddress": "localhost"
}
PARÂMETRO TIPO TAMANHO OBRIGATÓRIO DESCRIÇÃO
appToken ALFANUMÉRICO 64 SIM Identificação da aplicação
appPassword ALFANUMÉRICO 255 SIM Senha da aplicação
login ALFANUMÉRICO 255 SIM Nome do Usuário cadastrado na aplicação
userPassword ALFANUMÉRICO 255 SIM Senha do Usuário cadastrado na aplicação
ipAddress ALFANUMÉRICO 100 SIM IP do user que está acessando a aplicação
Retorno da Requisição AUTH - Privada
{
    "token": "FA9622016767FE46EA6601042586C5CD6D4AFB84723105E353DDDE07A55A8461",
    "user": {
        "firstname": null,
        "lastname": null,
        "name": "Vinicius Yago Bruno Pereira",
        "email": "heitorlucasferreira-99@delboux.com.br",
        "login": "heitorlucasferreira-99@delboux.com.br",
        "company": 1,
        "isactive": true,
        "phone": "",
        "role": "CUSTOMER"
    },
    "hasError": false,
    "error": null
}
PARÂMETRO TIPO TAMANHO OBRIGATÓRIO DESCRIÇÃO
appToken ALFANUMÉRICO 64 SIM Valor variável e retornado a cada requisição
firstname ALFANUMÉRICO 255 NÃO Será retornado se existir no cadastro
lastname ALFANUMÉRICO 255 NÃO Será retornado se existir no cadastro
name ALFANUMÉRICO 255 NÃO Será retornado se existir no cadastro
email ALFANUMÉRICO 255 SIM E-mail do cadastro
login ALFANUMÉRICO 255 SIM Se não for informado no cadastro, retornará o E-mail
company NUMÉRICO 50 SIM Código de identificação da empresa
isactive TEXTO 5 SIM Informativo sobre o Status da conta.
phone NUMÉRICO 50 NÃO Será retornado se existir no cadastro
role TEXTO 80 SIM Retornará o tipo do perfil do cadastro
haserror TEXTO 5 SIM Retornará "TRUE" se houver erros na requisição
error ALFANUMÉRICO 255 SIM Se houver erros, retornarão nesta chamada

O TOKEN gerado será válido por até 15 minutos, ou até a sua utilização. Após utilizá-lo, a API retornará os dados da sua requisição + um novo token para usar na próxima requisição.

POST

/auth/

Requisição AUTH - Pública

A requisição do tipo POST para o serviço de autenticação Pública(Auth) serve para criar uma autenticação via TOKEN para as requisições do tipo PÚBLICA. Após enviar essa requisição utilizando os parâmetros "AppToken" e "appPassword", outras ações na API serão permitidas para o TOKEN em questão, que será gerado e retornado após a requisição ser executada.

ATENÇÃO!!! Esse TOKEN será válido por até 15 minutos e somente poderá se utilizado uma ÚNICA vez, pois a cada requisição será gerado um novo TOKEN.

Requisição POST de exemplo
{
"appToken": "CDD3D0B1C293E96933F2311DF45E52446826C1FF3C7EEBD71076761BA5E9FF25",
"appPassword": "0253BBFC56DC68DB82AEE56127676B60B3F7F5DE2AF72D0FC974673A16313158"
}
PARÂMETRO TIPO TAMANHO OBRIGATÓRIO DESCRIÇÃO
appToken ALFANUMÉRICO 64 SIM Identificação da aplicação
appPassword ALFANUMÉRICO 64 SIM Senha da aplicação

/account/

Requisição Account

A requisição do tipo POST para o serviço de Account serve para cadastrar uma nova conta na aplicação. Nessa requisição existem diversos parâmetros opcionais e não obrigatório.

Requisição POST de exemplo
{
    "token": "611AAE9D6FE958FBED46A46BCEA8AA8CE7593B40E3E8853EC524D45C6C98706B",
    "ipAddress": "localhost",
    "doLogin": "false",
    "countryIsoCode": "BR",
    "fisrtname": "Cleiton",
    "lastname": "Kreimer",
    "name": "Cleiton",
    "login": "loginteste15",
    "password": "teste123",
    "email": "teste@gmailteste.com",
    "phone": "(49) 9 9999-9999",
    "taxid": "123.456.789-01",
    "address": "Av general osório",
    "addressCountryIsoCode": "BR",
    "addressRegionCode": "SC",
    "addressCityCode": "CCO",
    "addressDistrict": "Centro",
    "addressZipcode": "8980121",
    "accountNo": "",
    "sponsorAccountNo": "0550000000055"
}
PARÂMETRO TIPO TAMANHO OBRIGATÓRIO DESCRIÇÃO
token ALFANUMÉRICO 64 SIM Valor variável e retornado a cada requisição
ipAddress ALFANUMÉRICO 255 SIM IP do user que está acessando a aplicação
doLogin TEXTO 5 SIM Preencher com TRUE OR FALSE para efetuar login após o cadastro
countryIsoCode TEXTO 2 SIM Código do país. Usar BR ou PY
firstname TEXTO 255 NÃO Primeiro nome
lastname TEXTO 255 NÃO Último nome
name TEXTO 255 NÃO Nome completo
login ALFANUMÉRICO 255 NÃO Login usado para logar na aplicação. Caso não preenchido, será considerado o e-mail
password ALFANUMÉRICO 255 NÃO Senha para acesso a conta. Caso não seja preenchido, será considerado o e-mail
email ALFANUMÉRICO 255 SIM E-mail que será usado no cadastro
phone NUMÉRICO 50 NÃO Número de telefone
taxid NUMÉRICO 25 NÃO Número do CPF ou CEDULA
address ALFANUMÉRICO 255 NÃO Endereço
adressCountryisoCode TEXTO 2 NÃO Código do País do endereço
adressRegionCode TEXTO 2 NÃO Código da Unidade Federativa do endereço
adressCityCode TEXTO 3 NÃO Código da Cidade do endereço
adressDistrict TEXTO 255 NÃO Localidade do endereço
adressZipcode NUMÉRICO 15 NÃO CEP do endereço
accountNo NUMÉRICO 255 NÃO Deixar em branco
sponsorAccountNo NUMÉRICO 255 NÃO Código da conta de quem fez a indicação, se for o caso
Retorno da Requisição Account
{
    "token": "DC4521FA04504FC083C21EC1A155E8DF06FCAC7FF69030A0A803AF494B6DF419",
    "hasError": false,
    "error": null,
    "didLogin": false,
    "user": null
}
PARÂMETRO TIPO TAMANHO OBRIGATÓRIO DESCRIÇÃO
token ALFANUMÉRICO 64 SIM Valor variável e retornado a cada requisição
hasError TEXTO 5 SIM Retornará "FALSE" se não houver erros na requisição
error TEXTO 4 SIM Retornará null se não houver erros na requisição
didLogin TEXTO 5 SIM Retornará TRUE ou FALSE de acordo com o que foi enviado na requisição
user ALFANUMÉRICO 255 SIM Retornará null se no parâmetro "didLogin" for enviado FALSE.
